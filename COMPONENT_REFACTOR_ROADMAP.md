## 🎓 重构原则卡片

### 卡片 1：单一职责原则

```
┌─────────────────────────────────────┐
│ 单一职责原则 (SRP)                   │
├─────────────────────────────────────┤
│                                      │
│ ✅ 每个组件只做一件事                │
│ ✅ 每个 hook 只管理一种状态          │
│ ✅ 每个函数只完成一个功能            │
│                                      │
│ ❌ 避免"万能组件"                    │
│ ❌ 避免"上帝对象"                    │
│                                      │
└─────────────────────────────────────┘
```

### 卡片 2：组件大小限制

```
┌─────────────────────────────────────┐
│ 组件大小限制                         │
├─────────────────────────────────────┤
│                                      │
│ 🟢 小型：< 100 行（理想）            │
│ 🟡 中型：100-300 行（可接受）        │
│ 🟠 大型：300-500 行（需注意）        │
│ 🔴 超大：> 500 行（必须拆分）        │
│                                      │
└─────────────────────────────────────┘
```

### 卡片 3：逻辑分离

```
┌─────────────────────────────────────┐
│ 逻辑分离                             │
├─────────────────────────────────────┤
│                                      │
│ UI 展示    → 组件 (JSX)              │
│ 业务逻辑   → hooks (useXxx)          │
│ 计算工具   → utils (functions)       │
│ 类型定义   → types (interfaces)      │
│                                      │
└─────────────────────────────────────┘
```

### 卡片 4：命名规范

```
┌─────────────────────────────────────┐
│ 命名规范                             │
├─────────────────────────────────────┤
│                                      │
│ 组件：ProfileCard, ProfileList      │
│ Hook：useProfileData, useProfile    │
│ 工具：validateProfile, formatXxx    │
│ 类型：Profile, ProfileData          │
│                                      │
│ ❌ 避免：ProfileViewer, ProfileThing│
│                                      │
└─────────────────────────────────────┘
```
