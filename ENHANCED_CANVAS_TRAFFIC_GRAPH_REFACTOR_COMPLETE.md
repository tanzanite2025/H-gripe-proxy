## 🎓 重构经验总结

### 成功经验

1. **按职责拆分**
   - 先拆分渲染器（最独立）
   - 再拆分 hooks（业务逻辑）
   - 最后拆分 UI 组件
   - 最后重构主组件

2. **职责单一**
   - 每个渲染器只负责一种绘制
   - 每个 hook 只负责一种逻辑
   - 每个组件只负责一种展示

3. **保持简洁**
   - 主组件只负责组合
   - 渲染器只负责绘图
   - Hooks 只负责逻辑

4. **性能优化**
   - RAF 调度独立
   - 防抖处理独立
   - 焦点检测独立

### 遇到的问题

1. **问题：** RefObject 类型不匹配
   - **解决：** 使用 `RefObject<HTMLCanvasElement | null>`
   - **教训：** useRef 返回的类型包含 null

2. **问题：** 渲染器函数参数过多
   - **解决：** 使用对象参数或分组
   - **教训：** 超过 5 个参数应该重构

3. **问题：** Canvas 绘图逻辑复杂
   - **解决：** 按绘制内容拆分为多个渲染器
   - **教训：** 复杂绘图需要模块化
