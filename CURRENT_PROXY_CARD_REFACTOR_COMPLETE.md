## 🎓 重构经验总结

### 成功经验

1. **逐步拆分**
   - 先提取工具函数（最简单）
   - 再提取 hooks（业务逻辑）
   - 最后拆分 UI 组件
   - 最后重构主组件

2. **职责单一**
   - 每个文件只做一件事
   - 数据管理、UI 展示、业务逻辑分离
   - 工具函数独立

3. **保持简洁**
   - 主组件只负责组合
   - 子组件只负责展示
   - Hooks 只负责逻辑

4. **及时验证**
   - 每个阶段后运行类型检查
   - 发现问题立即修复
   - 避免累积错误

### 遇到的问题

1. **问题：** 工具函数文件使用了 JSX 但扩展名是 `.ts`
   - **解决：** 重命名为 `.tsx`
   - **教训：** 包含 JSX 的文件必须使用 `.tsx` 扩展名

2. **问题：** 导入路径包含 `.tsx` 扩展名导致类型错误
   - **解决：** 移除导入路径中的扩展名
   - **教训：** TypeScript 导入不应包含扩展名

3. **问题：** 延迟检测逻辑复杂，难以拆分
   - **解决：** 创建独立的 hook，使用 ref 管理状态
   - **教训：** 复杂逻辑需要独立的 hook
