## 📋 功能详解

### 1. IP 信息查询 🌐

**功能**:
- 显示当前 IP 地址
- 显示地理位置（国家、城市）
- 显示 ISP（互联网服务提供商）

**使用场景**:
- 查看当前 IP 信息
- 验证代理是否改变了 IP

**特点**:
- ✅ 国内服务优化，速度快
- ✅ 自动刷新（5 分钟）
- ✅ 手动刷新

---

### 2. 代理检测 🔒

**功能**:
- 检测代理是否生效
- 对比直连 IP 和代理 IP
- 智能识别代理特征

**使用步骤**:

#### 首次使用
1. **关闭代理**
2. 点击"保存为直连 IP"按钮
3. **启用代理**
4. 自动检测代理是否生效

#### 后续使用
- 启用代理后自动检测
- 查看检测结果：
  - ✅ **代理已生效** - IP 和位置都改变了
  - ⚠️ **未检测到代理** - 可能未启用或使用本地代理

**建议**:
- 首次使用时保存直连 IP
- 定期更新直连 IP（30 天有效期）
- 切换网络环境后重新保存

---

### 3. DNS 泄漏检测 🛡️

**功能**:
- 检测 DNS 服务器位置
- 对比 DNS 位置和代理位置
- 评估 DNS 泄漏风险

**风险等级**:
- ✅ **安全** - DNS 位置 = 代理位置
- ⚠️ **警告** - DNS 位置 ≠ 代理位置（中等风险）
- 🔴 **危险** - DNS 在国内，代理在国外（高风险）

**修复建议**:
- 启用 DNS over HTTPS (DoH)
- 使用 fake-ip 模式
- 配置代理 DNS

**使用场景**:
- 验证 DNS 是否泄漏
- 检查 DNS 配置是否正确
- 排查隐私泄漏问题

---

### 4. 速度测试 ⚡

**功能**:
- **下载速度测试** - 10MB 测试文件
- **上传速度测试** - 5MB 测试数据
- **延迟测试** - 10 次测试，计算 min/max/avg/jitter
- **丢包测试** - 20 次测试，统计丢包率

**使用步骤**:
1. 点击"开始测试"按钮
2. 等待测试完成（约 30-60 秒）
3. 查看测试结果

**测试流程**:
```
1. 下载测试 (10-20 秒)
   - 实时显示当前速度
   - 显示进度条

2. 上传测试 (5-10 秒)
   - 实时显示当前速度
   - 显示进度条

3. 延迟测试 (2-3 秒)
   - 10 次测试
   - 计算 min/max/avg/jitter

4. 丢包测试 (2-3 秒)
   - 20 次测试
   - 统计丢包率

5. 显示结果
```

**结果解读**:

| 指标 | 优秀 | 良好 | 一般 | 较差 |
|------|------|------|------|------|
| 下载速度 | >50 Mbps | 10-50 Mbps | 1-10 Mbps | <1 Mbps |
| 上传速度 | >20 Mbps | 5-20 Mbps | 1-5 Mbps | <1 Mbps |
| 延迟 | <50 ms | 50-150 ms | 150-300 ms | >300 ms |
| 丢包率 | <1% | 1-5% | 5-10% | >10% |

**注意事项**:
- 测试会消耗流量（约 15MB）
- 可随时点击"停止测试"中断
- 测试结果受网络环境影响

---

### 5. WebRTC 泄漏检测 🔐

**功能**:
- 检测 WebRTC 是否泄漏真实 IP
- 区分本地 IP 和公网 IP
- 评估泄漏风险

**风险等级**:
- ✅ **安全** - 未检测到泄漏
- ⚠️ **警告** - 泄漏本地 IP（风险较低）
- 🔴 **危险** - 泄漏公网 IP（风险高）

**修复建议**:

#### Chrome
1. 安装 WebRTC Leak Prevent 扩展
2. 或在 `chrome://flags` 中禁用 WebRTC

#### Firefox
1. 访问 `about:config`
2. 设置 `media.peerconnection.enabled = false`

#### Edge
1. 安装 WebRTC Control 扩展

**使用场景**:
- 验证 WebRTC 是否泄漏
- 检查浏览器配置
- 排查隐私泄漏问题

### 场景 2: 深度诊断
**目标**: 全面检查网络状态和安全性

**步骤**:
1. 访问"网络诊断"页面
2. 运行所有检测：
   - IP 信息
   - 代理检测
   - DNS 泄漏检测
   - WebRTC 泄漏检测
   - 速度测试

### 场景 3: 问题排查
**目标**: 排查网络或代理问题

1. 访问"网络诊断"页面

3. 识别问题：
   - 代理未生效？→ 检查代理配置

5. 重新测试验证

---

### 场景 4: 性能测试
**目标**: 测试代理节点性能

**步骤**:
1. 切换到要测试的代理节点

3. 运行速度测试
4. 查看结果：
   - 下载/上传速度
   - 延迟和抖动
   - 丢包率
   - 稳定性
5. 对比不同节点的性能
6. 选择最佳节点

## 🔧 故障排除

### 问题 1: 无法获取 IP 信息
**可能原因**:
- 网络连接问题
- 服务源不可用

**解决方案**:
1. 检查网络连接
2. 点击刷新按钮重试
3. 切换代理节点

---

### 问题 2: 代理检测显示"未检测到代理"
**可能原因**:
- 代理未启用
- 使用本地代理
- 未保存直连 IP

**解决方案**:
1. 确认代理已启用
2. 保存直连 IP
3. 重新检测

---

### 问题 3: DNS 泄漏检测失败
**可能原因**:
- 网络连接问题
- DNS 服务不可用

**解决方案**:
1. 检查网络连接
2. 点击刷新按钮重试
3. 等待几分钟后再试

---

### 问题 4: 速度测试失败
**可能原因**:
- 网络连接不稳定
- 测试服务不可用

**解决方案**:
1. 检查网络连接
2. 重新开始测试
3. 切换代理节点

---

### 问题 5: WebRTC 检测显示"浏览器不支持"
**可能原因**:
- 浏览器版本过旧
- WebRTC 已被禁用

**解决方案**:
1. 更新浏览器到最新版本
2. 检查 WebRTC 是否被禁用
3. 使用其他浏览器
