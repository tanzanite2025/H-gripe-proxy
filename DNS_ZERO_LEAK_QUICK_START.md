# DNS 零泄漏防护 - 快速开始

## 什么是 DNS 泄漏？

DNS 泄漏是指即使使用了代理或 VPN，DNS 查询仍然暴露给 ISP 或中间人，导致隐私泄露。

**风险**:
- ⚠️ ISP 可以看到你访问的所有网站
- ⚠️ 中间人可以监控你的网络活动
- ⚠️ DNS 污染和劫持

---

## 快速配置（3 步）

### 步骤 1: 打开设置页面

在 Clash Verge 中打开 **设置** 页面

### 步骤 2: 选择防护级别

在 **DNS 零泄漏防护** 卡片中选择防护级别：

| 级别 | 适用场景 | 延迟 | 隐私 |
|------|----------|------|------|
| 基础防护 ⭐ | 日常使用 | 30-50ms | 中 |
| 严格防护 | 强隐私需求 | 20-40ms | 高 |
| 偏执防护 | 高风险环境 | 30-80ms | 最高 |

**推荐**: 基础防护（平衡速度和隐私）

### 步骤 3: 测试 DNS 泄漏

点击 **开始测试** 按钮，验证配置是否生效

✅ 如果显示 "未检测到 DNS 泄漏"，说明配置成功！

## 防护级别对比

### 基础防护（推荐）⭐

```
✅ 强制使用 DoH 加密
✅ DNS 泄漏测试
✅ 基本隐私保护
⚡ 延迟: 30-50ms
```

**适合**: 日常使用、需要基本隐私保护

### 严格防护

```
✅ 启用 Fake-IP 模式
✅ 阻止明文 DNS
✅ 阻止系统 DNS
✅ 强制使用 DoH
⚡ 延迟: 20-40ms
```

**适合**: 需要强隐私保护、防止 DNS 污染

### 偏执防护

```
✅ 启用 Fake-IP 模式
✅ 阻止明文 DNS
✅ 阻止系统 DNS
✅ 阻止 IPv6 DNS
✅ 强制使用 DoH 和 DoT
⚡ 延迟: 30-80ms
```

**适合**: 高风险环境、对抗审查

## 验证 DNS 是否泄漏

### 方法 1: 内置测试（推荐）

1. 打开设置页面
2. 找到 "DNS 零泄漏防护" 卡片
3. 点击 "开始测试"
4. 查看测试结果

### 方法 2: 在线测试

1. 访问 https://dnsleaktest.com
2. 点击 "Extended test"
3. 查看 DNS 服务器列表
4. 确认所有 DNS 都是加密的（Cloudflare、Google 等）

### 方法 3: 命令行测试

```bash
# Windows
nslookup example.com

# Linux/Mac
dig example.com
```

## 常见问题

### Q: 启用后网速变慢怎么办？

A: DoH 加密会增加延迟，可以：
- 使用 "基础防护" 而不是 "偏执防护"
- 使用国内 DoH 服务器（阿里 DNS）
- 启用 DNS 缓存

### Q: 什么是 Fake-IP 模式？

A: Fake-IP 模式不进行真实 DNS 查询，直接返回虚假 IP，由 Clash 处理连接。优点是延迟最低、防护最强。

### Q: 如何判断 DNS 没有泄漏？

A: 
- ✅ 所有 DNS 查询都通过 DoH/DoT
- ✅ DNS 服务器不是 ISP 的服务器
- ✅ 没有明文 DNS 查询

## 技术原理

### DoH (DNS over HTTPS)

通过 HTTPS 加密 DNS 查询，ISP 无法看到查询内容

```
传统 DNS: 你的电脑 → ISP DNS → 网站
DoH: 你的电脑 → HTTPS 加密 → DoH 服务器 → 网站
```

### Fake-IP 模式

不进行真实 DNS 查询，直接返回虚假 IP

```
传统模式: 应用 → DNS 查询 → 获取 IP → 连接
Fake-IP: 应用 → 虚假 IP → Clash 拦截 → 真实连接
```
